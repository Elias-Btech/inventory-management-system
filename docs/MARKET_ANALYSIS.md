# Market Analysis — StockWise Pro

## Problem Statement

Small and medium businesses often rely on spreadsheets or paper-based systems to track inventory. These approaches are error-prone, hard to share across a team, and provide no real-time visibility into stock levels.

## Target Users

| Segment | Use Case |
|---------|----------|
| Small retail stores | Track product stock, record daily sales |
| Warehouses | Manage large inventories, monitor stock movements |
| Restaurants / cafes | Track ingredients and supplies |
| E-commerce businesses | Sync inventory with order fulfillment |
| University / academic projects | Demonstrate full-stack web development |

## Key Pain Points Addressed

- **No real-time stock visibility** → Dashboard with live stats and low stock alerts
- **No audit trail** → Activity logs track every action by every user
- **No access control** → Role-based system (Admin vs Staff)
- **Manual reporting** → Automated CSV exports and on-screen reports
- **Uncontrolled user registration** → Passcode system prevents unauthorized accounts

## Competitive Advantages

- Zero cost (open source, self-hosted)
- No internet dependency (runs on local server)
- Simple setup via XAMPP
- Fully customizable source code
- Clean, modern UI that works on mobile

## Technology Justification

| Choice | Reason |
|--------|--------|
| PHP | Widely supported, easy to deploy on shared hosting |
| MySQL | Industry standard, reliable relational database |
| Vanilla JS | No framework dependency, fast load times |
| XAMPP | Standard local dev environment for PHP projects |
